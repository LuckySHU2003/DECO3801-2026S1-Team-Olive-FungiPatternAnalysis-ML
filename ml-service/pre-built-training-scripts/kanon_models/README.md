## Explanation Doc
This system processes the time-series voltage data of the mushroom signals, and perform three machine learning models which includes Kmean and HMM for pattern detection and LSTM for prediction model.
1) Data Loading and Validation
- Load .csv and .xlsx file and assume time column is the first column and the rest of the columns are voltage columns. 
- Validate the dataset by checking the null values and the number of columns. Check whether the time column is valid datetime and voltage columns are numeric. 

2) Preprocessing of the Data
- Perform noise reduction by smoothing the signals using the moving average
- Perform baseline stabilisation by subtracting the rolling mean from the dataset which removes the slow drift in the data.

3) Extract Features
- The voltage is split into sliding window, and it extracts Frequency, Amplitude mean, Amplitude max, Spike interval mean and Spike interval std by using find_peak function.

4) Pattern Detection (Kmean)
- Cluster the data into 3 groups as there are three main features which are Frequency, Amplitude and Spike interval. 
- After the clustering each clusters are converted into meaningful patterns where high amplitude is labelled as spike, low interval is labelled as oscillation and others are labelled as baseline. 
- The frequency of each pattern type is computed and it returns the percentage distribution for each patterns. 
- The confidence score is calculated by the distance ratio between the closest and the second-closest cluster centroids. 

5) Pattern Detection (HMM)
- Convert the voltages to observations which includes the value, difference and the absolute difference 
- Train gaussianHMM  based on the observations and learn the hidden states which includes, spikes, oscillations and baselines
- Predict the hidden states
- The confidence score is calculated by the likelihood that the observed voltage was generated by the learned hidden states and the log-likehood is converted into a normalised confidence score. 


7) Time-filtered Analysis and Allowing Re-Run
- Adds time-filtering function to the existing hmm model
- It tracks the past runs into the history_log and able to compare the result with the past results. 

8) Prediction Model (LSTM)
- Uses K-fold training where the data is split into 5 parts and each parts will be trained, and the average of the predictions will be the final output
The confidence score is calculated by 1 – standard deviation across the predictions

## Project Design
Load and Clean Data 
- Remove noise such as humidity changes, mechanical vibrations 
- Methods like outlier detection, clustering by spike features (amplitude, width, symmetry), or comparing with known noise patterns can detect these noises so we can remove them.
- Normalise voltage values 
Detect Spikes 
- Identify all significant voltage changes 
- Segment spikes from background activity 
Extract Features 
- Amplitude 
- Width 
- Symmetry 
- Rise/fall dynamics (fast up/slow down, slow up/fast down) 
Pattern Recognition
- Use clustering (find groups of similar spikes) or HMM (find sequences of patterns and temporal dynamics)
- Identify recurring patterns and potentially new patterns 
Analyse Recurrence and Correlation 
- Frequency of each spike type 
- Relation to species, growth stage, or environmental conditions 
- Correlation between spike features such as amplitude vs width
Prediction 
- Predict next spike timing or shape 
- Start with simple models:  consider LSTM or temporal CNN for advanced prediction 
Visualise 
- Overview + zoom-in approach 
- Heatmaps for pattern frequency 
- Highlight patterns by color/type 
- Correlation plots with environmental variables 

Validate and Interpret 
- Check consistency across recordings/species 
- Assess biological meanings like synchronisation, metabolic activity
- Review rare/unexpected patterns
